# Block Reviews

One soft review per completed training block. **The Hevy training log is the source
of truth** for what was actually lifted (extracted via `scripts/hevy/block_report.py`);
the Google Sheets are the planning surface only. A review is written at the end of a
block — or, if missing, before the next block is designed (the `designing-training-block`
skill enforces this).

Each review covers four things, plus coach's notes on power and health:
1. **What's been done** — block summary, plan vs actual on the Big 5 (real numbers)
2. **What to continue** — what's working, keep it
3. **What to improve / change** — the honest part
4. **Action items** — concrete, carried into the next block
